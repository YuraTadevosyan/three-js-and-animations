# three-webgl-showcase

Interactive Three.js + WebGL showcase built with **React 18**, **TanStack Router & Query**, **Tailwind CSS**, and **shadcn/ui**. Tuned for SEO and Core Web Vitals.

## Stack

| Layer        | Choice                                     |
| ------------ | ------------------------------------------ |
| Build        | Vite 5 (es2020 target, Brotli + gzip)      |
| UI runtime   | React 18 + TypeScript (strict)             |
| Routing      | TanStack Router (file-based, code-split)   |
| Data         | TanStack Query                             |
| 3D / WebGL   | three + @react-three/fiber + @react-three/drei |
| Shaders      | Custom GLSL via Vite `?raw` imports        |
| Styling      | Tailwind CSS + tailwindcss-animate         |
| Components   | shadcn/ui primitives (`Button`, `Card`)    |
| Linting      | ESLint flat config + typescript-eslint     |
| Format       | Prettier                                   |
| SEO          | react-helmet-async, sitemap, robots, OG/Twitter, JSON-LD |

## Skills demonstrated

- **Three.js / WebGL**
  - r3f scenes with proper resource lifecycle
  - Custom GLSL vertex + fragment shader (`src/shaders/wave.*.glsl`)
  - Buffer geometry with 6k particles, single draw call
  - PBR materials with environment mapping
  - DPR clamping (`dpr={[1, 1.75]}`) and `powerPreference: 'high-performance'`
- **React patterns**
  - File-based, type-safe routing
  - Lazy-loaded scenes with `React.lazy` + `IntersectionObserver`
  - `prefers-reduced-motion` honored at CSS and animation-loop level
  - StrictMode-safe useFrame hooks
- **TanStack**
  - Router context typed against the QueryClient
  - `defaultPreload: 'intent'` for instant route transitions
  - Devtools loaded only in dev via dynamic import
- **Performance**
  - Manual chunks for `three`, r3f, and TanStack
  - Brotli + gzip pre-compression at build
  - Per-route code splitting via `autoCodeSplitting`
  - Canvas mounts only when scrolled into view
  - DPR clamps + dev-mode-only devtools
- **SEO**
  - Per-route `<title>`, description, canonical, Open Graph, Twitter cards
  - JSON-LD `WebSite` structured data in `index.html`
  - `robots.txt`, `sitemap.xml`, web manifest
  - Crawlable `<noscript>` fallback content
  - Skip-to-content link, semantic landmarks, `aria-live` loading states

## Deploy to GitHub Pages

The site is deployed under
`https://yuratadevosyan.github.io/three-js-and-animations/three-webgl-showcase/`,
sharing the `gh-pages` branch with sibling showcase projects (one subfolder
per project).

Key wiring:

- `vite.config.ts` sets `base: '/three-js-and-animations/three-webgl-showcase/'`
  so all asset URLs are prefixed correctly.
- `src/main.tsx` reads `import.meta.env.BASE_URL` into TanStack Router's
  `basepath`, so route matching strips the prefix automatically.
- `public/404.html` captures any deep link (GitHub Pages doesn't rewrite for
  SPAs) and redirects to `index.html?/<original-path>`. A snippet in
  `index.html` decodes that back into a real URL via `history.replaceState`
  before React boots — so /examples/sky works on a hard reload, not just a
  client-side navigation.

To deploy: `npm run deploy`. This builds and runs
`gh-pages -d dist -e three-webgl-showcase`, which publishes `dist/` into the
`three-webgl-showcase` subfolder of the `gh-pages` branch.

## SEO note for SPAs

Since this is client-rendered, the `<noscript>` block in `index.html` and the JSON-LD give crawlers something meaningful at the document level. For richer indexing of per-route metadata, deploy through a host that prerenders routes (Cloudflare, Netlify) or migrate to TanStack Start for SSR.
